    Description Logic

    Learning outcomes:

    After completing this lesson you should be able to:

    • Explain the meaning of the terms "concept" and "role" as used in description logics.
    • Write definitions of concepts in description logic using a variety of constructs for both concepts and roles.
    • Identify when characteristics of roles are needed, such as being transitive, functional, or symmetric.
    • Explain why the statement "Mary likes all cats" is possible to express in some description logics but not in others.

    2.1 Introduction

    2.1.1 The ingredients

    Description Logic exists to describe two kinds of thing:

    • Concepts
      For example: \(\mathsf{Human}\), \(\mathsf{Pizza}\), \(\mathsf{Grandmother}\), \(\mathsf{MeatTopping}\).

      • Think of a concept as a set of things (its instances).
      • The word 'concept' means the same as 'class' in this context.

    • Roles
      For example: \(\mathsf{hasTopping}\), \(\mathsf{isMotherOf}\), \(\mathsf{isGrandmotherOf}\), \(\mathsf{likesToEat}\).

      • Think of a role as a binary relation between things.

    2.1.2 Building concepts

    Concepts are built up in the following ways:

    • Atomic concepts
      These are names of concepts e.g. \(\mathsf{Human}\), \(\mathsf{Pizza}\).

    • Universal concept
      This is denoted by: \(\top\)
      Instances are all things in the ontology.

    • Bottom concept
      This is denoted by: \(\bot\)
      This is the class with no instances, also called the empty concept.
    • Negated concept
      This is denoted by: \(\neg C\) where \(C\) is any concept
      Instances are things not in \(C\).
    • Concept intersection
      This is denoted by: \(C \sqcap D\) where \(C\) and \(D\) are any concepts.
      The things in this class are those things that are instances of both \(C\) and of \(D\).
      Note that this is written \(\sqcap\) and not \(\cap\).
    • Concept union
      This is denoted by: \(C \sqcup D\) where \(C\) and \(D\) are any concepts. The things in this class are those things that are instances of \(C\) or of \(D\) (or both).
      Note that this is written \(\sqcup\) and not \(\cup\).
    • Universal value restriction
      This is denoted by: \(\forall r.C\) where \(C\) is a concept and \(r\) is a role.
      The things in this class are all those things where everything to which they stand in relation \(r\) belongs to the class \(C\).
      For example, to talk about the class of things that only eat vegetables we could say:
    \[ \forall \mathsf{eats} . \mathsf{Vegetable}. \]
    • Although the symbol \(\forall\) is the same as in first order logic, be careful
      that the way it is used here in the syntax is very different. In particular,
      \(\mathsf{eats}\) is not a variable. To understand the connection with first order logic
      imagine a language with two predicates \(\mathit{eats}\) and \(\mathit{Vegetable}\). The concept above would correspond to the set of those \(x\) for which the following holds
    \[ \forall y\, (\mathit{eats}(x,y) \to \mathit{Vegetable}(y)). \]
    • Existential value restriction
      This is denoted by: \(\exists r.C\) where \(C\) is a concept and \(r\) is a role.
      The things in this class are all those things where something to which they stand in relation \(r\) belongs to the class \(C\).
      For example, to talk about the class of things that eat at least one thing which is a vegetable we could say:
    \[ \exists \mathsf{eats} . \mathsf{Vegetable}. \]
    • Although the symbol \(\exists\) is the same as in first order logic, be careful
      that the way it is used here in the syntax is very different. In particular,
      \(\mathsf{eats}\) is not a variable. To understand the connection with first order logic
      imagine a language with two predicates \(\mathit{eats}\) and \(\mathit{Vegetable}\). The concept above would correspond to the set of those \(x\) for which the following holds:
    \[ \exists y\, (\mathit{eats}(x,y) \wedge \mathit{Vegetable}(y)). \]

    More examples of restriction

    Existential flavour of restriction

    Suppose we have a class: \(\mathsf{Mansion}\) and a property \(\mathsf{owns}\).

    Two individuals are related by this property just when the first owns the second.
    For example a particular person owning a specific car.

    We can build a new class from \(\mathsf{Mansion}\) and \(\mathsf{owns}\).

    This is the class consisting of every individual who \(\mathsf{owns}\) a \(\mathsf{Mansion}\).

    In Description Logic we could write \(\exists\, \mathsf{owns} \cdot \mathsf{Mansion}\)

    This means the class of everyone who owns a mansion,
    or people where something they own is a mansion.
    We could call this class \(\mathsf{Rich}\), or better say it's a subclass of \(\mathsf{Rich}\).

    In Protégé the same class is written:

    owns  some  Mansion
    Universal flavour of restriction

    Suppose we have a class: \(\mathsf{Worthless}\) and a property \(\mathsf{owns}\).

    We can build a new class from \(\mathsf{Worthless}\) and \(\mathsf{owns}\).

    This is the class consisting of every individual who only \(\mathsf{owns}\) things that are \(\mathsf{Worthless}\).

    In Description Logic we could write \(\forall\, \mathsf{owns} \cdot \mathsf{Worthless}\)

    This means the class of everyone who owns only worthless things,
    or people where everything they own is worthless.
    We could call this class \(\mathsf{Poor}\). It includes anyone owning nothing.

    In Protégé the same class is written:

    owns  only  Worthless

    Notice how this is different from:

    owns  some  Worthless

    More restriction examples

    More examples of the power of Description Logic to define new classes out of existing ones and properties.

    • Teaching staff are staff who teach some module:
    \[\mathsf{TeachingStaff} \equiv \mathsf{Staff} \sqcap \exists\, \mathsf{teaches} . \mathsf{Module}\]
    • A PG-only module is a module only studied by postgraduate students:
    \[\mathsf{PGOnlyModule} \equiv \mathsf{Module} \sqcap \forall\, \mathsf{studiedBy} . \mathsf{PostGrad}\]

    2.1.3 Number restrictions

    If \(R\) is a property (role) and \(n\) a non-negative integer then:

    \[\leq n \; R \quad\mathsf{~and~}\quad \geq n \; R \quad\mathsf{~and~}\quad = n \; R\qquad \mathsf{~are~concepts.}\]

    The concept \(\leq n \; R\) consists of all those entities having the \(R\) relationship to at most \(n\) things. The other two concepts have the same idea, except we would be talking about 'at least \(n\) things' or 'exactly \(n\) things'.

    For example, if we want to say a criminal with at least three cars is successful, we could do this:

    \[\mathsf{Criminal} \sqcap (\geq 3 \; \mathsf{hasCar}) \sqsubseteq \mathsf{SuccessfulCriminal}.\]

    A qualified number restriction is one of these:

    \[\leq n \; R . C \qquad \geq n \; R . C \qquad = n \; R . C\]

    These differ from the previous ones in having a concept \(C\) as well as a role \(R\). We can then talk about not simply having three cars, but having three cars which belong to some class. For example:

    \[\geq 3 \; \mathsf{hasCar} . \mathsf{Electric}.\]

    For another example, the concept of having exactly four bedrooms and all rooms (not just bedrooms) being spacious.

    \[(= 4 \; \mathsf{hasRoom} . \mathsf{BedRoom}) \sqcap \forall\, \mathsf{hasRoom}.\mathsf{Spacious}\]

    Note that \(\geq 1 \; R.C\) is equivalent to \(\exists \, R.C\)

    2.1.4 Sub-class relationships

    Terminological axioms allow us to state:

    • Concept equivalence
      This is written \(C \equiv D\)
      Asserting that means concepts have the same instances.

    • Concept subsumption This is written \(C \sqsubseteq D\).
      Asserting that means \(C\) is a sub-concept of \(D\).

    2.1.5 Building roles

    There are many different flavours of Description Logic and one way these differ is in the ways they allow you to construct roles out of other roles. We will see some of these constructions later, but to get started we only need the constructions on roles.

    2.2 TBox and ABox example

    This example shows how you can use the ways of constructing concepts to define some concepts from some given atomic ones.

    Description Logic Example: TBox

    \(\mathsf{Atomic ~Concepts ~and ~Roles:} \quad \left\{ \begin{array}{ll} \mathsf{Person} &\mathsf{(concept)} \\ \mathsf{Female} &\mathsf{(concept)} \\ \mathsf{hasChild}&\mathsf{(role)}\\ \end{array} \right.\)

    \(\begin{array}{rcl} \mathsf{Woman} & \equiv & \mathsf{Person} \sqcap \mathsf{Female} \\ \mathsf{Mother} & \equiv & \mathsf{Woman} \sqcap \exists \mathsf{hasChild}.\mathsf{Person} \\ \mathsf{Parent} & \equiv & \mathsf{Person} \sqcap \exists \mathsf{hasChild}.\mathsf{Person}\\ \mathsf{ParentOfOnlyDaughters} & \equiv & \mathsf{Person} \sqcap \forall \mathsf{hasChild}.\mathsf{Female} \\ \mathsf{Grandmother} & \equiv & \mathsf{Mother} \sqcap \exists \mathsf{hasChild}.\mathsf{Parent}\\ \end{array}\)

    A typical Description Logic specification consists of:

    • the TBox, which is a set of terminological axioms as in the above example, and also
    • the ABox, which is a set of assertions about individuals as in the following.

    Description Logic example: ABox

    \(\qquad\qquad\begin{array}{l} \mathsf{Woman}(\mathsf{MARGE})\\ \mathsf{Woman}(\mathsf{LISA})\\ \mathsf{hasChild}(\mathsf{MARGE}, \mathsf{LISA}) \end{array}\)

    This example indicates the basic ideas, but there is a lot that you might want to say about the concepts and roles. For example:

    • Every person has exactly two people whose child they are.
    • The \(\mathsf{hasChild}\) role cannot relate anyone to themself or to their own parents.

    We shall see how to express these later.

    2.3 Characteristics of relationships

    2.3.1 Introduction

    A relationship means a way in which individuals might be related,
    for example:

    • Parent of, child of, brother of, sister of, ancestor of, etc.
    • Older than, younger than, taller than, heavier than, etc.
    • Studies, teaches, studies with.
    • Is inside, is next to, overlaps with, etc.
    • Is part of, has part.
    • Earlier than, later than, at same time as, during, etc.
    • Author of, editor of, illustrator of.
    • Actor in, director of, script writer of.
    • Was written in, was first screened.
    • Follows, is followed by, knows (person), is friend of, etc.

    Unfortunately terminology is not uniform. The following three words are used to mean the same thing by different communities:

    • Relationship (this is used in Entity-Relationship modelling).
    • Property (this is used in OWL, the Web Ontology Language in Protégé).
    • Role (this is used in many Description Logics).

    A relationship can have features (such as being transitive, or being reflexive). As OWL uses the word property already we will say characteristics instead of properties for these features. This is what Protégé does as you can see in the following screenshot:

    The seven characteristics listed are all used in the usual sense of these words in books and other resourses on discrete mathematical structures. The characteristic inverse functional is often called injective in a lot of settings, and also gets called one-to-one in some places.

    2.3.2 When to use characteristics

    Before we review the definitions of the characteristics, it is important to note when it is appropriate to say that a relationship has one of these characteristics in an ontology:

    • The relationship 'ancestor of' has the property:
      • If A is an 'ancestor of' B, and B is an 'ancestor of' C
        Then A is an 'ancestor of' C.
    • This is not about:
      • accidental features of particular individuals that happen to be in an ontology at one time.
    • This is about
      • all individuals that could ever possibly be related.

    Compare that with the relationship 'has heard of' between people (used in the usual sense of the phrase, and applied to a large number of people):

    • it might just happen that we have an ontology in which it is true of all the individuals there at present that
      • if A has 'heard of' B, and B has 'heard of' C,
        then A has 'heard of' C;
    • but this does not mean we should be asserting that 'heard of' has the same characteristic as 'ancestor of' does. We only assert that a relationship has a characteristic if it is forced to have it by virtue of the meaning of the relationship as it can apply to any individuals whatsoever.

    2.3.3 The seven characteristics

    • A relationship \(R\) is functional means:
      if \(A\) is any individual, then it never happens that

      • there are individuals \(B\) and \(C\) which are different from each other, and
      • \(R(A, B)\) and \(R(A,C)\),

    • A relationship \(R\) is inverse functional means:
      if \(A\) is any individual, then it never happens that

      • there are individuals \(B\) and \(C\) which are different from each other, and
      • \(R(B, A)\) and \(R(C, A)\).

    • A relationship \(R\) is transitive means:
      if \(A\), \(B\), \(C\) are any individuals
      (including possibly some of them being equal to others)
      then it always happens that

      • if \(R(A,B)\) and \(R(B,C)\) then \(R(A,C)\).

    • A relationship \(R\) is symmetric means:
      if \(A\) and \(B\) are any individuals
      (including possibly both being the same)
      then it always happens that

      • if \(R(A,B)\) then \(R(B,A)\).

    • A relationship \(R\) is asymmetric means:
      if \(A\) and \(B\) are any individuals
      (including possibly both being the same)
      then it never happens that

      • both \(R(A,B)\) and \(R(B,A)\).

    • A relationship \(R\) is reflexive means:
      if \(A\) is any individuals
      then it always happens that

      • \(R(A,A)\) holds.

    • A relationship \(R\) is irreflexive means:
      if \(A\) is any individuals
      then it never happens that

      • \(R(A,A)\) holds.

    All of these definitions need to be read using the language in a very literal way, just as in the way logic is used.

    Beware of these points:

    • The characteristic irreflexive is not the same as 'not reflexive'. A relation can be not reflexive without being irreflexive.
    • The characteristic asymmetric is not the same as 'not symmetric'. A relation can be not symetric without being asymmetric.
    • The key part of the definition of transitive says: 'If \(R(A,B)\) and \(R(B,C)\)'. In the event that these do not both hold for particular \(A\), \(B\), \(C\), or that there are no individuals at all for which both hold, neither of these things does anything to prevent \(R\) being transitive (remember how implication works in propositonal logic).
    • The key part of the definition of symmetric also uses 'if' at a similar key point.

    2.4 Other features of and constructions with roles

    2.4.1 Domain and range

    It is possible to say that individuals connected by a property must belong to specified classes.

    We might want to say that a property \(\mathsf{teaches}\) should be from the class \(\mathsf{Staff}\) to class \(\mathsf{Module}\).

    In this case \(\mathsf{Staff}\) is the Domain and \(\mathsf{Module}\) is the Range.

    The Domain and the Range can be the same, but this is NOT the same as the property being reflexive!

    One Description Logic notation for domain and range is:

    • \(\mathsf{Domain}(R,C)\).
      Meaning that whenever \(R\) relates individual \(x\) to individual \(y\), then \(x\) belongs to \(C\).
    • \(\mathsf{Range}(R,C)\).
      Meaning that whenever \(R\) relates individual \(x\) to individual \(y\), then \(y\) belongs to \(C\).

    2.4.2 Sub-property and super property

    We know:
    If \(A \text{ is } \mathrel{\mathsf{motherOf}} B\) then \(A \text{ is } \mathrel{\mathsf{parentOf}} B\).

    In Description Logic:
    The \(\mathsf{motherOf}\) property is a sub-property of \(\mathsf{parentOf}\) and we write this as:

    \[\mathsf{motherOf} \sqsubseteq \mathsf{parentOf}.\]

    Another way of saying the same thing is:
    \(\mathsf{parentOf}\) property is a super property of \(\mathsf{motherOf}\).

    In Description Logic we use \(\sqsubseteq\) for inclusion of properties as well as for classes.

    2.4.3 Composing properties (chains)

    In Description Logic, OWL and Protégé (but not in lots of other places):

    Given properties \(R\) and \(S\),
    \(R \circ S\) is the property where \(x\) is linked to \(z\) if:

    1. \(x\) is linked to something \(y\) by \(R\), and
    2. \(y\) is linked to \(z\) by \(S\)

    This is nice for pictures but opposite to the most popular (if arguably misguided) convention for function composition.

    Chain composition example

    Classes: Module, Student, Degree

    Properties: takenBy, compulsoryFor, studiedBy

    • A module can be studied by a student.
    • A degree can be taken by a student.
    • A module can be compulsory for a degree.

    How do we say this?
    - If a module is compulsory for a degree taken by a student then the module must be studied by the student.

    \[\mathsf{compulsoryFor} \circ \mathsf{takenBy} \sqsubseteq \mathsf{studiedBy}\]

    In Protégé: \(\mathsf{studiedBy}\) needs to be a super property of the chain \(\mathsf{compulsoryFor} \circ \mathsf{takenBy}\)

    The composition or chain \(\mathsf{compulsoryFor} \circ \mathsf{takenBy}\) is a new property relating every module which is compulsory for some degree to every student taking that degree.

    Composition is a powerful idea. See how it works in this exercise.

    Composition exercises

    Assuming parentOf, childOf, motherOf and daughterOf have conventional meanings, which of these correspond to or include the properties granddaughterOf, grandmotherOf, siblingOf, grandparentOf, or other easily named things?

    1. childOf o childOf.
    2. parentOf o childOf.
    3. childOf o parentOf.
    4. daughterOf o childOf.
    5. motherOf o parentOf.
    6. parentOf o brotherOf.

    Some more examples

    Concept:
    Knowing someone who has a child who plays football:

    \[\qquad \exists \; (\mathsf{knows} \circ \mathsf{hasChild}) . \mathsf{PlaysFootball}\]

    concept:
    Having a child who knows someone who plays football:

    \[\qquad \exists \; (\mathsf{hasChild} \circ \mathsf{knows}) . \mathsf{PlaysFootball}\]

    \[\qquad \exists y \; \exists z\; (\mathsf{knows}(x,y) \wedge \mathsf{hasChild}(y,z) \wedge \mathsf{PlaysFootball}(z))\]

    How in Description Logic could we formulate the concept of:
    knowing someone who plays football who has a child?

    2.4.4 Boolean operations

    If \(R\) and \(S\) are roles then we can construct these roles:

    1. \(R \sqcup S\) (union)
      Two things are related by this role if they are related by \(R\) or by \(S\) or by both.

    2. \(R \sqcap S\) (intersection)
      Two things are related by this role if they are related by both \(R\) and \(S\).

    3. \(\neg R\) (complement)
      Two things are related by this role if they are not related \(R\).

    2.4.5 Transitive closure

    If \(R\) is a role then we can construct these roles:

    1. \(R^+\) (transitive closure).
      Two things \(x\) and \(y\) are related by this role if there is a sequence of \(n+1\) things

      \[ x_0, x_1, \ldots, x_n\]

      where \(x = x_0\), \(x_n = y\), \(n \geqslant 1\) and \(R(x_i, x_{i+1})\) for all \(0 \leqslant i < n\).

    2. \(R^\ast\) (reflexive transitive closure).
      Two things \(x\) and \(y\) are related by this role if there is a sequence of \(n+1\) things

      \[ x_0, x_1, \ldots, x_n\]

      where \(x = x_0\), \(x_n = y\), \(n \geqslant 0\) and \(R(x_i, x_{i+1})\) for all \(0 \leqslant i < n\).

    The difference between \(R^\ast\) and \(R^+\) is \(R^\ast(x,x)\) always holds, but \(R^+(x,x)\) only holds in the case that \(R(x,x)\).

    2.4.6 Identity and inverse

    If \(R\) is a role then we can have these roles:

    1. \(R^{-1}\) or \(R^-\) (inverse / converse).
      This is the role where \(R^-(x,y)\) iff \(R(y,x)\).

    2. \(\mathsf{id}(C)\) (identity role on concept \(C\)).
      This is the role where the domain and range are \(C\) and \(x\) is related to \(y\) iff \(x = y\).

    2.5 Expressivity of Description Logics

    2.5.1 The variety of different logics

    There are many different systems called description logics and they differ among themselves according to the different constructs allowed. We have introduced a variety of features to indicate what is possible, but not all description logics allow all the features we have discussed.

    A widely used starting point is a language denoted by \(\mathcal{AL}\). This only allows the formation of concepts as follows:

    • atomic concepts
    • the top, \(\top\), and bottom, \(\bot\), concepts
    • negation, \(\neg\), applied only to atomic concepts
    • intersection of concepts: \(C \sqcap D\)
    • universal restriction: \(\forall R . C\)
    • existential restriction in the limited form \(\exists R . \top\)

    In this logic unions of concepts are not allowed and expressions like \(\neg (C \cap D)\) are also not allowed. There are no constructions on roles, just atomic names for roles. In the literature on description logics there is a convention about naming logics by using particular letters in the name to indicate what features are allowed. For example, extending the logic \(\mathcal{AL}\) by allowing negation for arbitrary concepts results in a logic called \(\mathcal{ALC}\) where the \(\mathcal{C}\) stands for 'complement'.

    2.5.2 Mary likes all cats

    The statement 'Mary likes all cats' has become part of the culture almost as much as 'Tweety' being a bird when discussing non-monotontic logic, or the names 'Alice' and 'Bob' for opposed players in games.

    The significance of Mary liking all cats is that in several description logics, including \(\mathcal{ALC}\), it is impossible to express the concept of liking all cats, and hence there is no way to say that Mary belongs to this concept. To be precise, it is impossible to do this in terms of:

    • a role \(\mathsf{likes}\), and
    • concept \(\mathsf{Cat}\).

    Using these two things we can build the concept \(\forall\; \mathsf{likes} . \mathsf{Cat}\). This is not the concept of liking all cats. This is the concept of liking only cats. An individual who fitted the concept \(\forall\; \mathsf{likes} . \mathsf{Cat}\) might only like one thing. That one thing would certainly have to be a cat, but there might be any number of cats that the individual did not like.

    If we have a more expressive description logic where roles can be negated, then it is possible to construct the concept \(\forall\; \neg \mathsf{likes} . \neg \mathsf{Cat}\). Saying Mary belongs to the class defined by this concept is to say

    • Mary only dislikes non-cats

    or, in more natural but less concise English:

    • Everything that Mary does not like is not a cat.

    This is equivalent to saying Mary likes all cats. Because if we have any cat, then Mary cannot dislike it. So it has to be one of the things that Mary likes. Don't forget that this is quite different from saying Mary only likes cats. If Mary likes all cats, then there may be things other than cats which Mary likes.

    There is always a trade-off between expressivity and the abilty to have reasoning tools that compute using a particular language. In some cases adding constructs makes reasoning impossible to do efficiently, that is in a reasonable amount of time. In other cases, the inclusion of certain features actually makes some reasoning problems impossible altogether.

    2.5.3 What "Description Logic" means here.

    In this module the term "Description Logic" without any further qualification will allow the use of any of the constructs mentioned in this unit.

    If an exercise, question, or assessment requires the use of a particular description logic in which only certain constructs are allowed, then this will be stated explicitly.

    2.5.4 Comparison with RDFS

    In lesson 1 of this unit we already met the idea that RDFS, is very limited in what it can express. The example there was that we cannot say the classes of cats and of people are disjoint. In description logic this is simply

    \[\mathsf{Cat} \sqcap \mathsf{Person} \sqsubseteq \bot\]

    Some other examples of statements that are known to be impossible to make in RDFS are the following. These examples are found in R. Arp, B. Smith, A. D. Spear: 'Building ontologies with Basic Formal Ontology' MIT Press, 2015, (page 155).

    • Every person has a mother who is also a person
    • Every person has exactly two biological parents

    You should be able write these in description logic using the constructs we have seen in this lesson.

    2.6 Wording and representation

    These examples show English phrases corresponding to concepts and statements in Description Logic. Note in particular that the word 'and' can indicate different things depending on the context.

    Undergraduate students are students:

    \[\mathsf{UnderGrad} \sqsubseteq \mathsf{Student}\]

    Modules and students are disjoint:

    \[\mathsf{Module} \sqcap \mathsf{Student} \equiv \bot\]

    Every student is a postgrad or undergrad (or possibly both):

    \[\mathsf{Student} \sqsubseteq \mathsf{UnderGrad} \sqcup \mathsf{PostGrad}\]

    Undergrads and postgrads are students (beware the and here modelled by \(\sqcup\)):

    \[\mathsf{UnderGrad} \sqcup \mathsf{PostGrad} \sqsubseteq \mathsf{Student}\]

    People who are staff and students are postgrads (notice and modelled by \(\sqcap\)):

    \[\mathsf{Staff} \sqcap \mathsf{Student} \sqsubseteq \mathsf{PostGrad}\]

    or more accurately:

    \[\mathsf{Person} \sqcap \mathsf{Staff} \sqcap \mathsf{Student} \sqsubseteq \mathsf{PostGrad}\]

    but:

    Staff and students are people:

    \[\mathsf{Staff} \sqcup \mathsf{Student} \sqsubseteq \mathsf{Person}\]

    Exercise

    Here are some atomic concepts (classes) and roles (properties).

    \[\begin{array}{l} \mathsf{Animal}\\ \mathsf{Insect} \\ \mathsf{Human} \\ \mathsf{LivingThing} \\ \mathsf{Carnivore} \\ \mathsf{Herbivore} \\ \mathsf{Plant} \\[3ex] \mathsf{eats} \\ \mathsf{isEatenBy}\\ \end{array} \]

    Express in Description Logic the following statements:

    1. Animals are living things

    2. Plants and animals are living things.

    3. Nothing is both a plant and an animal.

    4. 'Herbivore' means 'animal that only eats plants'.

    5. 'Carnivore means 'living thing that eats animals (but may eat other things as well).

    Select to reveal answer 1
    \[\mathsf{Animal} \sqsubseteq \mathsf{LivingThing}\]
    Select to reveal answer 2
    \[\mathsf{Animal} \sqcup \mathsf{Plant} \sqsubseteq \mathsf{LivingThing}\]
    Select to reveal answer 3
    \[\mathsf{Animal} \sqcap \mathsf{Plant} \equiv \bot\]
    Select to reveal answer 4
    \[\mathsf{Herbivore} \equiv \mathsf{Animal} \sqcap \forall \mathsf{eats} . \mathsf{Plant}\]
    Select to reveal answer 5
    \[\mathsf{Carnivore} \equiv \mathsf{LivingThing} \sqcap \exists \mathsf{eats} . \mathsf{Animal}\]
