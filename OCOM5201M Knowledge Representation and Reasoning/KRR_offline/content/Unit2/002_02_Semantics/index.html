    Lesson 2: The semantics of first order logic

    \(\def\phi{\varphi} % this is needed to make the Greek letter phi work\)

    Learning outcomes for lesson 2

    In this lesson you will:

    • understand the concept of an interpretation
    • work with examples of interpretations
    • learn how to evaluate the truth value of sentences in the language
    • learn basic logical equivalences

    Introduction

    In propositional logic, an interpretation of the symbols in the language just means saying whether each propositional variable is true or false.

    In first order logic, an interpretation has to provide a lot more:

    • a set of things to be the individuals that the language is talking about,
    • elements of this set for all the constants in the language,
    • \(n\)-ary relations on this set for each of the \(n\)-ary relation symbols,
    • functions on the set for any function symbols in the language.

    An expression in a first order language is just a string of symbols that
    follow certain rules. It does not by itself mean anything. However suggestive the names chosen may be, an expression by itself really means nothing.

    Giving the symbols things that they refer to is called giving an interpretation for the language. This extends the idea of giving values to propositions in propositional logic.

    Once an interpretation is given, we can talk about whether a formula is true in that interpretation. Having a notion of truth, we can talk about formulas being valid, meaning true in all interpretations.

    In propositional logic, an interpretation just needs to give a truth value for each atomic proposition. There are only a finite number of ways to do this, and all of them can be summarised in a truth-table for the formula.

    In first order logic each interpretation has to provide a set of individuals, as well as meanings for the constants and the relation symbols in the language.

    This means there are infinitely many interpretations, so we cannot tell if something is true in all interpretations by checking all of them.

    Informal idea of an interpretation

    To specify an interpretation you must:

    • Give a non-empty set, \(D\), called the domain.

    • For each constant symbol, give the element of \(D\) for it.

    • For each \(n\)-ary relation symbol, give an \(n\)-ary relation on \(D\).

    • For each propositional symbol, give a truth value.

    In this module we use this simple notion of interpretation. There are variants, all of which would change what is said later about first order logic. These include allowing an empty domain, and the possibility of giving types to individuals so that there are multiple domains, one for each type. There are good reasons for these ideas, but we use the simplest approach here.

    The formal definition of 'interpretation' is given later in this lesson. First we look at some examples to motivate the idea.

    The domain

    There is no restriction on what the elements of \(D\) are.

    • They can be abstract things such as numbers, symbols, mathematical structures such as sets, etc.

    • They can be physical concrete things in the real world, provided there is no ambiguity about how such things are described.

    • They can be concepts used in the world such as a time, again provided there are unambiguous ways of referring to them.

    Examples of interpretations

    Let's use this first order language:

    • Constants: \(\mathit{Leeds}\), \(\mathit{Reggie}\)
    • Unary relation symbols: \(\mathit{owl}\), \(\mathit{city}\)
    • Binary relation symbols: \(\mathit{livesIn}\)

    We can make the domain \(D = \{1, 2, 3\}\), and specify:

    \[ \begin{array}{rcl} I(\mathit{Leeds}) & = & 1\\[1ex] I(\mathit{Reggie}) & = & 2\\[1ex] I(\mathit{city}) & = & \{1\}\\[1ex] I(\mathit{owl}) & = & \{2\}\\[1ex] I(\mathit{livesIn}) & = & \{(2,1)\} \end{array} \]

    You might wonder where Reggie has come from. The city of Leeds has a coat of arms with two owls and a Latin motto "Pro Rege et Lege", and it is tempting to misinterpret this as naming the two owls, one of which feels like it should be Reggie (Rege).

    Equality

    The relation \(=\) is important and you can include this as binary relation symbol in a first order language.

    We usually insist that \(=\) is interpreted as the equality relation on the domain, and we then talk about first order logic with equality.

    If the domain is \(\{1, 2, 3\}\), the interpretation of \({=}\) looks like the following diagram. In the diagram the arrows pointing from each number to itself should really be labelled by \({=}\) in each case.

    The diagram above is a visualisation of an interpretation. The diagram features a sketch of an oval on its side. The oval contains three numbers – one, two and three. Each of the numbers is topped by a clockwise arrow that circles upwards and then points back towards the number it came from.

    The binary relation of equality is reflexive, symmetric and transitive. It is also well behaved with respect to substitution.

    This means, for example, that if \(P\) is a unary relation symbol, and \(t_1 = t_2\) holds then \(P(t_1)\) and \(P(t_2)\) must always have the same truth value. However, writing \(P(t_1) = P(t_1)\) is not allowed. This is because the two expressions related by \(=\) are formulas and not terms.

    Truth and validity

    Given an interpretation we can calculate a truth value for some formulas in the language. For example, the formula \(\mathit{Leeds} = \mathit{Reggie}\) is true in example 2, because the domain element used to interpret the constant symbol \(\mathit{Leeds}\) is the same domain element as the one used to interpret \(\mathit{Reggie}\).

    In example 1 these domain elements are different, so the formula \(\mathit{Leeds} = \mathit{Reggie}\) is false in that case.

    We can 'extend' the interpretation \(I\), so that \(I(\mathit{Leeds} = \mathit{Reggie})\) is true if \(I(\mathit{Leeds})\) and \(I(\mathit{Reggie})\) are the same element of \(D\).

    Formulas with variables, like \(I(\mathit{livesIn}(\mathit{Reggie}, x))\), need the idea of a valuation.

    The informal idea of a valuation

    Given an interpretation \(I\) with domain \(D\), a valuation means a function that gives an element of \(D\) for each variable.

    For example, for the interpretation \(I\) in example 1, we can have a valuation \(V\) where \(V(x) = 1\) and \(V(y) = 2\).

    In this valuation the formula \(\mathit{livesIn}(y, x)\) can be evaluated to true, because \((V(y), V(x)) \in I(\mathit{livesIn})\).

    But \(\mathit{livesIn}(x, y)\) would be evaluated to false because \((V(x), V(y)) \notin I(\mathit{livesIn})\).

    Free and bound variables

    In simple arithmetic with natural numbers we might come across a statement such as

    \[x + 1 = 2x.\]

    This on its own, as a statement, is neither true nor false because we have not said how to understand \(x\).

    • Is the statement saying that the equality holds for all values of \(x\)?

    • Is the statement saying that it is possible to find some \(x\) making it true?

    • Is the statement in a context where a particular value of \(x\) is known, and the statement asserts that for this value the equality holds?

    • Or maybe it is not a statement at all. Maybe it is an instruction to find one or all values of \(x\) making it true.

    In first order logic, any variable which is not quantified in an explicit way is said to be a free variable. Variables which are associated to quantifiers are called bound variables. In all the following four examples every \(x\) is a free variable and every \(y\) is a bound variable.

    • \(\forall y\; (r(x,y)\vee r(y,x))\)
    • \(p(x) \to \exists y\; r(y,x)\)
    • \(\forall y\; (y = y)\)
    • \(x = x\)

    Being free or bound is not really a property of a variable itself, but of a particular occurrence of a variable in a formula. The same variable can appear bound and free in different places in the same formula. We shall try to avoid tricky examples like this next one but it illustrates that we need to be careful.

    \[p(y,y) \wedge \forall y \; \exists z (r(z,y) \to \exists y \; s(y,x))\]

    In that example

    • the two \(y\,\)s in \(p(y,y)\) are both free
    • the \(y\) in \(r(z,y)\) is bound by the universal quantification \(\forall y\)
    • the \(y\) in \(s(y,x)\) is bound by the existential quantification \(\exists y\)

    The example is logically indistinguishable from

    \[p(y,y) \wedge \forall w \; \exists z (r(z,w) \to \exists v \; s(v,x))\]

    in which the \(y\) bound to the universal quantifier has been replaced by \(w\) and the \(y\) bound to the existential quantifier has been replaced by \(v\).

    In many cases it is safe to replace bound variables by different named variables but we would not be able to replace all the bound \(y\,\) by \(z\,\)s in the example as the \(y\) originally bound by the \(\forall y\) gets captured by the \(\exists z\).

    The informal idea of truth

    Given a formula with no free variables and an interpretation, we can work out a truth value for the formula.

    The definition is recursive, breaking down the truth value of a formula in terms of simpler formulas.

    For example, if \(\alpha\) and \(\beta\) are formulas, then the definition shows how to calculate the truth value of \(\alpha \wedge \beta\) in terms of the truth values of \(\alpha\) and \(\beta\).

    Formal definition of interpretation

    Given a first order language, \(\mathscr{L}\), an interpretation for \(\mathscr{L}\) is a pair \((D, I)\) where

    • \(D\) is a non-empty set, called the domain, and
    • \(I\) is a function which:
      • to each constant symbol \(c\) of \(\mathscr{L}\) assigns an element \(I(c) \in D\),
      • to each \(n\)-ary predicate symbol \(P\), (\(n > 0\)), assigns an \(n\)-ary relation \(I(P) \subseteq D^n\),
      • to each propositional constant \(q\), assigns a truth value \(I(q)\).

    Sometimes \(I\) is called a structure.

    Valuations defined formally

    Given a structure \((D, I)\) for a language \(\mathscr{L}\), a valuation is a function from the set of all variables to \(D\).

    Given a valuation \(\mathcal{V}\), a variable \(x\), and \(d \in D\), we define the valuation \(\mathcal{V}[x \mapsto d]\) to be the same as \(\mathcal{V}\) for every variable other than \(x\), and \(x\) gets sent to \(d\).

    \[ \mathcal{V}[x \mapsto d]\, (u) = \left\{ \begin{array}{ll} \mathcal{V}(u) & \text{ if $u$ is not $x$,}\\ d & \text{ if $u$ is $x$.} \end{array} \right. \]

    If you imagine \(\mathcal{V}\) as a table of input-output pairs, this is the idea of updating one of these pairs.

    Remember that a term is either a variable or a constant symbol.

    Given \(\mathcal{A} = (D, I)\), and a valuation \(\mathcal{V}\), it is convenient to use the notation \(\mathcal{V}(t)\) when \(t\) is any term as follows

    \[ \mathcal{V}(t) = \left\{ \begin{array}{ll} \mathcal{V}(u) & \text{ if $t$ is a variable $u$,}\\[2ex] I(c) & \text{ if $t$ is a constant symbol $c$.} \end{array} \right. \]

    Definition of truth

    For a structure \(\mathcal{A} = (D, I)\), and a valuation \(\mathcal{V}\), and a formula \(\phi\), the notation

    \[\mathcal{A}, \mathcal{V} \vDash \phi\]

    will mean that \(\phi\) is true in the structure \(\mathcal{A}\) under the valuation \(\mathcal{V}\).

    This relationship is defined as follows in the cases that \(\phi\) is of the form \(P(t_1, \ldots, t_n)\) for an \(n\)-ary predicate symbol \(P\), or \(\phi\) is \(q\) for a propositional constant \(q\).

    \[\begin{array}{ll} \mathcal{A}, \mathcal{V} \vDash P(t_1, \ldots, t_n) & \text{ iff } (\mathcal{V}(t_1),\ldots, \mathcal{V}(t_n)) \in I(P)\\[1ex] \mathcal{A}, \mathcal{V} \vDash q & \text{ iff } I(q) \end{array}\]

    For formulas \(\alpha\) and \(\beta\) we define

    \[\begin{array}{ll} \mathcal{A}, \mathcal{V} \vDash \alpha \wedge \beta & \text{ iff both } \mathcal{A}, \mathcal{V} \vDash \alpha \text { and } \mathcal{A}, \mathcal{V} \vDash \beta \\[1ex] \mathcal{A}, \mathcal{V} \vDash \alpha \vee \beta & \text{ iff } \mathcal{A}, \mathcal{V} \vDash \alpha \text { or } \mathcal{A}, \mathcal{V} \vDash \beta \text{ or both}\\[1ex] \mathcal{A}, \mathcal{V} \vDash \alpha \to \beta & \text{ iff } \mathcal{A}, \mathcal{V} \vDash \alpha \text { does not hold, or } \mathcal{A}, \mathcal{V} \vDash \beta \text{ does, or both} \\[1ex] \mathcal{A}, \mathcal{V} \vDash \alpha \leftrightarrow \beta & \text{ iff } \mathcal{A}, \mathcal{V} \vDash \alpha \text { and } \mathcal{A}, \mathcal{V} \vDash \beta \text{ either both hold or neither does} \\[1ex] \mathcal{A}, \mathcal{V} \vDash \neg \alpha & \text{ iff it is not the case that } \mathcal{A}, \mathcal{V} \vDash \alpha\\[3ex] \mathcal{A}, \mathcal{V} \vDash \forall x\, \alpha & \text{ iff for every } d \in D \text{ it holds that } \mathcal{A}, \mathcal{V}[x \mapsto d] \vDash \alpha\\[1ex] \mathcal{A}, \mathcal{V} \vDash \exists x\, \alpha & \text{ iff for some } d \in D \text{ it holds that } \mathcal{A}, \mathcal{V}[x \mapsto d] \vDash \alpha \end{array}\]

    We can use \(\mathcal{A}, \mathcal{V} \nvDash \alpha\) to indicate that \(\mathcal{A}, \mathcal{V} \vDash \alpha\) does not hold.

    The turnstile notation

    The symbol \(\vDash\) is sometimes called 'double turnstile', and it has a counterpart 'single turnstile', \(\vdash\), which we meet later. The symbol \(\vDash\) is used in some ways that can be confusing when compared with the usage \(\mathcal{A}, \mathcal{V} \vDash \phi\) above.

    Remember that a sentence is a formula with no free variables.

    \(\vDash \phi\) means that for all structures, \(\mathcal{A}\), and all valuations \(\mathcal{V}\) for that structure, \(\mathcal{A}, \mathcal{V} \vDash \phi\).

    When \(\phi\) is a sentence, \(\phi\) will get the same truth value under all valuations. However we still need to deal with valuations even for sentences because of the way they appear in the definition of truth for quantified statements.

    Writing \(\mathcal{A} \vDash \phi\) will mean \(\phi\) is true in the structure \(\mathcal{A}\) (under any valuation). We then say \(\mathcal{A}\) is a model of \(\phi\).

    For formulas \(\phi\) and \(\psi\) we say \(\psi\) logically implies \(\phi\) if for every stucture and valuation such that \(\mathcal{A}, \mathcal{V} \vDash \psi\), it is also the case that \(\mathcal{A}, \mathcal{V} \vDash \phi\). Logical implication is denoted \(\psi \vDash \phi\).

    This notation can be confusing because we use both \(\mathcal{A} \vDash \phi\) and \(\psi \vDash \phi\) with very different entities on the left of the turnstile. There is no problem here, but be sure you are aware of the difference between a formula and an interpretation or a structure.

    If \(\Gamma\) is a set of sentences, we write \(\Gamma \vDash \phi\) to mean every model for all of the sentences in \(\Gamma\) is also a model of \(\phi\).
