    Lesson 1: Introducing first order logic

    In propositional logic there is only one kind of thing: statements (propositions). These can be true or false. The logic provides ways of building up complex statements out of simpler ones. But propositional logic is not expressive enough for many things we want to represent and reason about. First order logic, also called predicate logic or predicate calculus, is more expressive because as well as statements that can be true or false, there are also individual things for the statements to talk about.

    Lesson 1 Learning outcomes:

    After completing this lesson you should be able to:

    • distinguish relations and relation symbols (and so on)
    • propose language elements to represent given scenarios
    • distinguish between terms and formulas
    • use quantifiers and other parts of first order logic to write syntactically correct formulas.

    What first order logic adds to propositional logic

    In first order logic we can represent much more of the following situation than we could in propositional logic:

    • All of the red parcels were delivered yesterday.
    • The parcel with the new component was one of the red ones.
    • Every parcel delivered in the last 10 days was received undamaged.

    A consequence of those statements is:

    • The new component was received undamaged

    The novelties here (not in propositional logic) are:

    • things (e.g. parcels) in addition to propositions and statements,
    • properties things may have (a thing may be a parcel; it may be red),
    • relationships between things (a particular parcel contains certain things),
    • The words 'all', 'every' and 'one of' are used to talk about collections of individuals.

    To work with the statements we would also need a way of representing things such as:

    • Yesterday is within the last 10 days
    • If a parcel contained something and was received, then the contents were received.

    Relations, predicates, properties

    It can be confusing that many different words are often used to talk about the same things.

    Consider these examples:

    • Manchester is between Liverpool and Leeds
    • Sally is older than Fred
    • Sally is tall
    • It is true that it is raining

    We might represent these in first order logic by:

    • \(\mathit{between}(\mathit{Manchester}, \mathit{Liverpool}, \mathit{Leeds})\)
    • \(\mathit{olderThan}(\mathit{Sally}, \mathit{Fred})\)
    • \(\mathit{tall}(\mathit{Sally})\)
    • \(\mathit{itIsRaining}\)

    In this unit, we use the following terminology:

    • \(\mathit{between}\) is a relation symbol of arity 3; it takes 3 arguments
    • \(\mathit{olderThan}\) is a relation symbol of arity 2; it takes 2 arguments
    • \(\mathit{tall}\) is a relation symbol of arity 1; it takes 1 argument
    • \(\mathit{itIsRaining}\) is a relation symbol of arity 0; takes 0 arguments

    However, you might come across alternative conventions elsewhere. For example, people often:

    • say predicate symbol instead of relation symbol
    • drop the word 'symbol' if the difference between a relation and a name for a relation is obvious or unhelpful
    • say property instead of predicate or relation for arity 1
    • say proposition symbol or propositional constant for a predicate of arity 0
    • call things of arity 0 constants
    • call things of arity 1 unary
    • call things of arity 2 binary

    Parcel delivery example

    This gives an example of how the earlier statements about parcels can be represented in first order logic. This is only one possibility. Different options include having \(\mathit{Undamaged}\) as a unary relation symbol, rather than as a constant. The treatment of time is very simple and would not necessarily extend to different examples.

    \(\forall x\, (\mathit{parcel}(x) \wedge \mathit{red}(x) \to \mathit{delivered}(x, \mathit{Yesterday}))\)

    \(\exists x\,(\mathit{parcel}(x) \wedge \mathit{red}(x) \wedge \mathit{contains}(x, \mathit{NewComponent}))\)

    \(\forall x\, (\mathit{parcel}(x) \wedge \mathit{delivered}(x, \mathit{InLast10Days}) \to \mathit{received}(x, \mathit{Undamaged}))\)

    \(\forall x\, (\mathit{delivered}(x, \mathit{Yesterday}) \to \mathit{delivered}(x, \mathit{InLast10Days}))\)

    \(\forall x\, \forall y\, ((\mathit{contains}(x,y) \wedge \mathit{received}(x, \mathit{Undamaged})) \to \mathit{received}(y, \mathit{Undamaged}))\)

    and we should be able to deduce:

    \(\mathit{received}(\mathit{NewComponent}, \mathit{Undamaged}))\)

    The language

    The language is the basis of the syntax of our logic. What is described is only one possibility for the syntax of first order logic.

    It is made out of what are called symbols (these do not always have to be single characters - a word can be a 'symbol').

    Symbols come in two flavours:

    • logical, and
    • non-logical.

    The non-logical symbols depend on the application, the logical ones are always the same.

    Certain strings of these symbols will be valid expressions.

    There are two kinds of expressions: terms and formulas. Terms can refer to individuals, whereas formulas (also called formulae) model statements about individuals.

    Logical symbols

    These are the logical symbols:

    • Punctuation: parentheses and commas: ( and ) and ,

    • Connectives: \(\vee\), \(\wedge\), \(\neg\), \(\to\)

    • Quantifiers: \(\forall\), \(\exists\)

    • Variables: an infinite supply of these, usually written \(x\), \(y\) etc and sometimes \(x_1\), \(x_2\), etc.

    Non-logical symbols

    The non-logical symbols consist of

    • Constant symbols

      For specific individual things. For Example: \(\mathit{Yesterday}\).

      There is no fixed convention, but I have tried to use upper case (capital) letters at the start of constants to help distinguish them from relation symbols.

    • Relation symbols

      each with specified arity (no. of inputs \(\geqslant 1\))

      For relationships and properties. For Example: \(\mathit{between}\), \(\mathit{red}\).

    • Proposition symbols

      (\(0\)-ary relation symbols)

      For Example: \(\mathit{itIsRaining}\), etc

    Terms

    Terms are linguistic entities that can refer to individual things. Don't confuse terms with formulas, which can have truth values.

    To give an analogy, in English 'the cat is happy' is a sentence. Within that sentence 'the cat' refers to a particular individual thing in the world. This is like being a term in first order logic. The sentence as a whole is something that is true or false. This sentence is like a formula.

    The rules for building terms are very simple:

    • Every variable is a term.
    • Every constant symbol is a term.
    • Nothing else is a term.

    If we add function symbols this becomes slightly more complicated. Function symbols are an additional complication which are more useful for modelling aspects of mathematical practice than for most applications of knowledge representation in AI.

    To give an example, we can model one person being the mother of another in two ways. (For the purposes of the example about functions, we have to imagine that everybody has exactly one mother.)

    • as a relationship that holds between two individuals, or
    • as a function that given a person as input, returns their mother as output.

    The first of these approaches is more flexible. In the second approach we are forced to make sense of

    'the mother of the mother of the mother of … the mother of someone'

    where there is no limit at all to the number of generations hidden in the … .

    Formulas

    Formulas are things in the language that can have truth values.

    Don't confuse formulas with terms, which can refer to individual things.

    Formulas are also sometimes called well formed formulas or wffs, just as in the case of propositional logic.

    The rules for formulas:

    • Every propositional symbol is a formula.
    • If \(P\) is a predicate symbol of arity \(n \geqslant 1\), and \(t_1, \ldots, t_n\) are terms, then \(P(t_1, \ldots, t_n)\) is a formula.
    • If \(\alpha\) and \(\beta\) are formulas then these are formulas:
      • \(\neg \alpha\)
      • \((\alpha \vee \beta)\)
      • \((\alpha \wedge \beta)\)
    • If \(x\) is a variable, and \(\alpha\) is a formula, then \(\forall x \; \alpha\) and \(\exists x \; \alpha\) are formulas.
    • Nothing else is a formula.

    Note that the definition of what it means to be a formula is recursive.

    The informal idea to have at the moment is that

    \[ \left. \begin{array}{l} \forall x \;\alpha\\[1ex] \exists x \;\alpha\\[1ex] \neg \alpha\\[1ex] (\alpha \vee \beta)\\[1ex] (\alpha \wedge \beta)\\[1ex] (\alpha \to \beta) \end{array} \right\} \text{ means } \left\{ \begin{array}{l} \text{For all } x \text{ it is that case that } \alpha\\[1ex] \text{There is some } x \text{ such that } \alpha\\[1ex] \text{Not } \alpha\\[1ex] \alpha \text{ or } \beta\\[1ex] \alpha \text{ and } \beta\\[1ex] \alpha \text{ implies } \beta \end{array} \right. \]

    Beware that

    • 'or' is inclusive,
    • 'some' means one or more,
    • 'all' really means literally all, and not all except one or two.

    Parentheses get left out if there is no danger of ambiguity.

    Do not confuse terms with formulas

    You can imagine someone saying:

    When was the paint delivered? Not yesterday.

    You might be tempted to represent that by \(\mathit{delivered}(\mathit{ThePaint}, \neg \mathit{Yesterday})\), turning the informal 'not yesterday' into the formal \(\neg \mathit{Yesterday}\).

    This does not work. You can only use the logical connectives with statements that are capable of being true or false.

    The word \(\mathit{Yesterday}\) stands for an individual thing (a specific time), not something that can be true or false. You need to say \(\neg \mathit{delivered}(\mathit{ThePaint}, \mathit{Yesterday}).\)

    Check your understanding

    • Constant symbols : \(\mathit{Homer}\), \(\mathit{Bart}\), \(\mathit{Pink}\), \(\mathit{Yellow}\), \(\mathit{TheDonut}\)

    • Relation symbols (arity 1): \(\mathit{isAPerson}\), \(\mathit{isAColour}\), \(\mathit{isEdible}\)

    • Relation symbols (arity 2): \(\mathit{hasColour}\), \(\mathit{likes}\).

    • Proposition symbols: \(\mathit{itIsTuesday}\), and where

    • \(x\), \(y\), \(z\) are variables.

    Which of the following are formulas according to the rules?

    1. \(\mathit{isAColour}(\mathit{Pink}) \wedge \mathit{hasColour}(\mathit{TheDonut}, \mathit{Yellow})\)

    2. \(\forall x \; (\mathit{likes}(\mathit{Homer}, x) \vee \neg \mathit{isEdible}(x) )\)

    3. \(\forall x \; \neg \exists y \; (\mathit{hasColour}(x,y) \wedge \neg \mathit{isAColour}(y))\)

    4. \((\mathit{hasColour}(\mathit{Pink},\mathit{Yellow}) \vee \mathit{Homer}(\mathit{likes},\mathit{TheDonut}))\)

    5. \(\mathit{itIsFriday}\)

    6. \(\mathit{hasColour}(\mathit{isEdible}(x), \mathit{isAColour}(y))\)

    7. \(\forall x\; ( \exists y\; (\forall z\; \mathit{isEdible}(y) \wedge \mathit{likes}(z,z)) ) \vee \mathit{isEdible}(x))\)

    8. \((((\mathit{itIsTuesday} \wedge \neg \mathit{itIsTuesday}) \wedge \neg \neg \mathit{itIsTuesday}) \wedge \forall x \; \mathit{isEdible}(y))\)

    Reveal answers

    1, 2, 3 are all formulas.

    4 is not, because

    • \(\mathit{Homer}\) is not a binary relation symbol
    • \(\mathit{likes}\) is not a term

    5 is not, because

    • \(\mathit{itIsFriday}\) is not in the language at all.

    6 is not, because

    • \(\mathit{isEdible}(x)\) is a formula and there needs to be a term in its place
    • \(\mathit{isAColour}(y)\) is a formula and there needs to be a term in its place

    7 is not, because the parentheses are not in matching pairs

    8 is a formula.

    Remember that being a formula is simply about the syntax. Something can be a formula without corresponding to anything useful that you could imagine anyone wanting to represent in the first place.

    Equality

    In first order logic with equality there is a special binary relation symbol, \(=\).

    This is written in the infix style: we write \(x = y\) rather than \(=(x,y)\).

    The \(=\) sign is always interpreted as meaning equality in the usual sense between the individual things that the language is talking about. For example, \(x=x\) will always be a true statement. But if we have a binary relation symbol \(\mathit{greaterThan}\) the situation is different. If we want it to be the case that nothing is greater than itself, making \(\neg \mathit{greaterThan}(x,x)\) true, then we have to say explicitly that this holds. The key point here is that the sign \(=\) is the only relation or predicate symbol which always comes with a particular fixed meaning.

    In general we will always include \(=\) as part of the language in first order logic. If there is any exercise or example in which it is not to be used then this will be stated very clearly.
