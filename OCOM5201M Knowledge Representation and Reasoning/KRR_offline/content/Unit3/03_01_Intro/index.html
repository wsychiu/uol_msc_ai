    Introduction to proof

    Learning outcomes:

    After completing this lesson you should be able to:

    • explain the difference between semantic entailment and proof

    • explain the concepts of soundness and completeness

    • use the modus ponens rule to build up simple proofs as trees.

    The idea of proof

    We have seen the notion of semantic consequence, or entailment, denoted by \(\vDash\). We now come to a detailed examination of the notion of proof, although we have already met this in using Prover9 in unit 2.

    Proof concerns a relationship between formulas, usually of one or more formulas being related to a single formula. This relationship is denoted by the turnstile symbol \(\vdash\). For example, in propositional logic, we can say

    \[ A,\; A \to B \vdash B \]

    We can read this as saying that from the propositions \(A\) and \(A\to B\) we can prove the formula \(B\). There are many alternative ways of saying this: from \(A\) and \(A \to B\) we can deduce \(B\).

    So far this explanation is very informal. To be more precise we would need to specify which of the many different systems of proof we are talking about. Before we get to seeing a particular system of proof in detail we can still appreciate how proof differs from semantic consequence.

    The relationship

    \[ A,\; A \to B \vDash B \]

    holds. Following the description of semantic consequence in unit 1 we can easily check this by truth tables. It does not only hold for atomic propositions \(A\) and \(B\); it will hold for any propositional formulas \(\alpha\) and \(\beta\) that

    \[ \alpha,\; \alpha \to \beta \vDash \beta \]

    By thinking of the possible truth values of the formulas \(\alpha\) and \(\beta\), however complicated they may be, we can see that whenever \(\alpha\) and \(\alpha \to \beta\) are true then \(\beta\) must be true too.

    Proof is very different from this. The relationships that hold by virtue of a system of proof for propositional logic are ones that are constructed by rules that use only the syntactic structure of formulas. This is different from using truth tables and relating things based on the semantic meaning of formulas.

    Here is an example of such a rule. This one, which goes back many centuries, is called modus ponens. The way the rule works is that if \(\alpha\) and \(\alpha \to \beta\) are formulas that have been already proved, or which we are taking as input formulas in a process to generate new formulas, then we can add \(\beta\) to the things we have proved. The rule is given a label \(\mathsf{MP}\), for Modus Ponens, because later we have to deal with many different rules and we may want to keep track of which rule has been used.

    \[ \begin{prooftree} \AXC{$\alpha$} \AXC{$\alpha \to \beta$} \RightLabel{$\; \mathsf{MP}$} \BIC{$\beta$} \end{prooftree} \]

    Using this rule we can build up proofs as trees like this one:

    \[ \begin{prooftree} \AXC{$B$} \AXC{$A$} \AXC{$A \to (B \to C)$} \RightLabel{$\; \mathsf{MP}$} \BIC{$B \to C$} \RightLabel{$\; \mathsf{MP}$} \BIC{$C$} \end{prooftree} \]

    This justifies

    \[ A, B, A \to (B \to C) \vdash C \]

    There are many different systems of proof. Some have advantages that they make it simpler to understand the properties of proof itself, and others make it simpler to build proofs and to see how these correspond to ways of reasoning that humans perform. Other systems of proof are especially useful for implementing software that can generate proofs.

    In this unit we meet:

    • natural deduction: a system of proof that suits human reasoning. It has a large number of different rules.
    • resolution: a system of proof that suits implementation. This has a single rule, and is the basis for both Prover9 and Prolog.

    One of the reasons that we need to study both \(\vdash\) and \(\vDash\) is that in first order logic there is simply no way to examine all possible interpretations that make a set of formulas true. Unlike propositional logic we cannot just look at all the different cases as in a truth table; there is no way to examine all the possible domains that an interpretation might have.

    The relationship between \(\vdash\) and \(\vDash\) is, however, really important.

    • A particular notion of proof is said to be sound if whenever the \(\vdash\) relationship holds then the \(\vDash\) relationship holds. This will mean, in particular, that anything that can be proved is actually true. Soundness is an essential requirement in most situations.
    • A particular notion of proof is said to be complete if whenever the \(\vDash\) relationship holds then the \(\vdash\) relationship holds. This will mean, in particular that anything which is true can actually be proved. For a lot of useful logical theories completeness is impossible to achieve. For example, Gödel famously showed (very roughly speaking) that if you write down some first order statements and include some very basic properties of the natural numbers in these statements, then there will always be formulas which are true in any model of your statements but which cannot be proved.
