    Modal logic

    \(\def\HA{\mathit{HoldsAt}}\) \(\def\Past{\mathop {\mathbf{P}}}\) \(\def\Future{\mathop {\mathbf{F}}}\) \(\def\Going{\mathop {\mathbf{G}}}\) \(\def\HasBeen{\mathop {\mathbf{H}}}\) \(\def\mBox{\mathop{\Box}}\) \(\def\mDiamond{\mathop{\Diamond}}\) \(\def\leq{\leqslant}\) \(\def\geq{\geqslant}\) \(\def\phi{\varphi}\)

    Learning outcomes:

    After completing this lesson you should be able to:

    • Construct simple examples of frames and valuations for propositional modal logic.

    • Evaluate the truth of modal logic formulas in a given interpretation.

    • Explain how some expressions in description logic are essentially formulas in modal logic.

    • Write first order formulas that correspond to modal logic formulas in some special cases where this is possible.

    3.1 Syntax of modal logic

    In this lesson, we will study propositional modal logic. There are various kinds of first order modal logics, but the propositional case is the most relevant here. Although it is called propositional it is capable of expressing some things that are not possible in ordinary propositional logic, and in some ways even beyond first order logic.

    Description logics are one example of modal logics being used in knowledge representation. Another example, we meet later in this unit are tense logics. These are logics for expressing things like:

    • In the future there will be pelicans in the park.
    • At every future time, the values of \(x\) and \(y\) in the program execution will be equal.
    • At some point in the past people lived here and there was a canal but not a railway.

    We start with a simple form of modal logic that captures the main idea, but which is too simple to fit the way that description logics are examples of modal logic.

    3.1.1 The language

    A language for propositional modal logic consists of:

    • Propositional variables, that stand for atomic propositions just as in ordinary propositional logic

    • Logical connectives as before: \(\quad \neg \quad \vee \quad \wedge \quad \to \quad \leftrightarrow\)

    • Two additional logical symbols:

      • \(\Box\), called box, and
      • \(\lozenge\), called diamond.

    • The usual parentheses to avoid ambiguity.

    3.1.2 The formulas

    The formulas of modal logic are expressions that can be true or false. Unlike first order logic there are no terms in the language that can refer to individual things.

    The rules for construction of formulas are:

    • Every atomic proposition is a formula

    • If \(\alpha\) and \(\beta\) are formulas then the following are all formulas:

      \[\neg \alpha \qquad \alpha \vee \beta \qquad \alpha \wedge \beta \qquad \alpha \to \beta \qquad \alpha \leftrightarrow \beta \qquad \Box \alpha \qquad \lozenge \alpha\]

    Parentheses can be added as necessary.

    The precedence rules follow those of first order logic with the box and diamond having the same precedence that the quantifiers have there.

    3.1.2.1 Examples

    If \(A\), \(B\), \(C\) are atomic propositions then the following are all examples of formulas in propositional modal logic:

    \[ \begin{array}{l} A\\[2ex] \neg B\\ (A \to B) \leftrightarrow C\\[2ex] \Box A\\[2ex] \lozenge B\\[2ex] (\Box A) \vee \neg A\\[2ex] \Box (A \vee \neg A)\\[2ex] (\Box \lozenge \Box \lozenge \neg A) \wedge ((\Box \Box B) \to \lozenge (B \vee \Box C)) \end{array} \]

    3.2 Semantics

    3.2.1 Informal idea

    In ordinary propositional logic, when we give the meaning of an atomic proposition we simply say whether it is true or false. In modal logic we can think of propositions not simply being true or false, but being true at some places and false at others. The idea of place here is not necessarily anything to do with space, it's more a way of thinking about how the semantics works. The 'places' might be different times, or objects in the world.

    The semantics for modal logic has two ingredients that go beyond semantics for propositional logic. These are:

    • set, often denoted \(W\), which models the idea of "places" where propositions can be true, and
    • a binary relation on \(W\), often denoted \(R\), so that \(R \subseteq W \times W\).

    Why use \(W\)?

    The letter 'W' is chosen because the elements of the set are sometimes referred to as worlds. This comes from the philosophical history of modal logic where the logic was used to talk about statements being true in two senses:

    • in all possible worlds, or
    • in some possible worlds.

    For example, it could be claimed that \(2+2=4\) is necessarily true (assuming the meanings of the symbols involved) and could not possibly be otherwise. On the other hand we can imagine a world in which the number \(4\) is commonly denoted by a different symbol (as of course it is in various cultures).

    In a lot of computer science applications, 'worlds' does not fit the context and they are instead called states.

    Modal logic is similar to first order logic in having this set of entities, but in the modal logic we study (at first) there is no way to refer to individuals in this set.

    The relation, \(R\), is often called the accessibility relation and this models the idea that some of the 'places' are accessible from a given 'place'.

    • A simple example is if \(W\) models a set of times, and the relation \(R\) models a relation of being later among the times.
    • The accessibility relation can also model a role as in description logic. For example, the 'worlds' might be people and the accessibility relation could model the idea of being a parent.

    In modal logic, unlike propositional logic and first order logic, there is a kind of local truth. That is, a formula can be true at one particular world but it may not be true at all worlds.

    3.2.2 Formal definition of semantics

    Let's have the formal definition and then come back to some examples.

    Given a language as described above, an interpretation provides:

    • frame, that is a pair \((W, R)\) where

      • a set \(W\)
      • a binary relation \(R\subseteq W \times W\),

    • a valuation \(V\) which provides for each atomic proposition, \(p\), a subset of \(W\) so \(V(p) \subseteq W\).

    then the semantics is:

    \[ \begin{array}{lcl} \mathcal{M}, s \vDash p & \mathsf{ iff } & s \in V(p)\\ \mathcal{M}, s \vDash \neg \alpha & \mathsf{ iff } & \mathcal{M}, s \nvDash \alpha\\ \mathcal{M}, s \vDash \alpha \vee \beta & \mathsf{ iff } & \mathcal{M}, s \vDash \alpha \mathsf{~or~ } \mathcal{M}, s \vDash \beta\\ \mathcal{M}, s \vDash \alpha \wedge \beta & \mathsf{ iff } & \mathcal{M}, s \vDash \alpha \mathsf{ ~and~ } \mathcal{M}, s \vDash \beta\\ \mathcal{M}, s \vDash \alpha \to \beta & \mathsf{ iff } & \mathcal{M}, s \nvDash \alpha \mathsf{~or ~} \mathcal{M}, s \vDash \beta\\ \mathcal{M}, s \vDash \lozenge \alpha & \mathsf{ iff } & \mathsf{There~ exists~ } t \in W \mathsf{~such ~that~} R(s,t) \mathsf{~and~} \mathcal{M}, t \vDash \alpha\\ \mathcal{M}, s \vDash \Box \alpha & \mathsf{ iff } & \mathsf{For ~ every ~} t \in W \mathsf{~such~that~} R(s,t), \mathsf{~it~holds~that~} \mathcal{M}, t \vDash \alpha\\ \end{array} \]

    The key points to understand are that (for a given valuation):

    • \(\lozenge p\) is true at \(w\) when \(p\) is true somewhere accessible from \(w\)

    • \(\Box p\) is true at \(w\) when \(p\) is true everywhere accessible from \(w\)

    This means that:

    • \(\lozenge\) has some similarities with \(\exists\) in first order logic.

    • \(\Box\) has some similarities with \(\forall\) in first order logic.

    3.2.3 Truth, satisfaction and validity

    In ordinary propositional logic, we said a formula \(\phi\) was satisfiable if it was true under some valuation, and valid if it was true under all valuations.

    In propositional modal logic, a formula \(\phi\) can be:

    • True at a particular world in a frame \(F\) under valuation \(\mathcal{V}\).

    • True at all worlds in a frame \(F\) under valuation \(\mathcal{V}\) (satisfied in \(F\) by \(\mathcal{V}\)).

    • True at all worlds in a frame \(F\) under all valuations (valid in \(F\)).

    • Valid in all frames (universally valid).

    In propositional modal logic, the notion of satisfiable (able to be true) for a formula \(\phi\) could be confusing. In the above terminology able to be true and able to be satisfied are used in different ways. This means we may need to be careful to distinguish these cases.

    3.4 Exercises

    Universally valid formulas

    These formulas are all valid in any frame, whatever the accessibility relation, and whatever the valuations for the propositional variables.

    • \(\qquad p \vee \neg p\)

    • \(\qquad \lozenge \lozenge (p \vee \neg p) \to \lozenge (p \vee \neg p)\)

    • \(\qquad \lozenge (p \vee \neg p) \vee \Box (p \wedge \neg p)\)

    • \(\qquad \Box (p \to q) \to (\Box p \to \Box q)\)

    To make sure you understand how modal logic works you should see if you can explain why each one has to be valid in all situations.

    You may find it helpful to think in terms of pictures with arrows like the ones drawn above. In pictorial terms we can understand when certain formulas would be true at a particular state. Taking one step is to be understood as moving along one arrow in the accessibility relation. The accessibility relation is allowed to have loops, so taking one step might not lead anywhere new.

    • From here you can take one step: \(\lozenge (p \vee \neg p)\).
    • From here you can take two steps: \(\lozenge \lozenge (p \vee \neg p)\).
    • From here you cannot go anywhere: \(\Box (p \wedge \neg p)\).

    Formulas that cannot be true everywhere

    The following two formulas are not universally valid.

    • \(\qquad p \wedge \neg p\)

    • \(\qquad p \wedge \lozenge \neg p\)

    The first formula is never going to be true at any state at all, however you choose a valuation for any frame.

    The second one is never going to be true at all states in any situation.

    • Can you explain why this must be so for \(p \wedge \lozenge \neg p\)?

    • Can you find a simple example of a frame and a valuation where \(p \wedge \lozenge \neg p\) holds at some states but not others?

    • Can you find an example of a frame where \(p \wedge \lozenge \neg p\) is always false at all states under all possible valuations?

    Formulas that are valid in some but not all frames

    For each of the following formulas you should be able to:

    • find examples of particular frames where the formula is valid. That means true at every state and whatever the valuation is, and
    • find examples of frames which do not have all those properties.

    In each case drawing simple examples can be helpful.

    • \(\qquad \lozenge \lozenge p \to \lozenge p\)

    • \(\qquad p \to \lozenge p\)

    • \(\qquad \Box p\)

    3.5 Comparison with first order logic

    In some cases modal logic and first order logic are just different ways of doing the same thing. In other cases there are significant differences in what can be represented in each.

    If we have a first order language with unary predicates \(\mathit{Red}\,\) and \(\mathit{Green}\) then in any interpretation we can say that certain formulas represent given statements. Here are some examples:

    • Something is red: \(\qquad \exists x \; \mathit{Red}(x)\).

    • Everything is green: \(\qquad \forall x\; \mathit{Green}(x)\).

    • Everything is either red or green: \(\qquad \forall x\; (\mathit{Green}(x) \vee \mathit{Red}(x))\)

    • More than one thing is red: \(\qquad \exists x \exists y \; (\mathit{Red}(x) \wedge \mathit{Red}(y) \wedge \neg (x = y))\).

    With an additional binary predicate \(\mathit{atLeastAsBig}\) we can express things like:

    • Any red thing is at least as big as all the green things:

    \(\qquad \forall x \forall y\;((\mathit{Green}(x) \wedge \mathit{Red}(y)) \to \mathit{atLeastAsBig}(y,x))\)

    • 'At least as big as' has the transitive property:

    \(\qquad \forall x \forall y \forall z\;((\mathit{atLeastAsBig}(x,y) \wedge \mathit{atLeastAsBig}(y,z)) \to \mathit{atLeastAsBig}(x,z))\)

    • Everything is at least as big as itself:

    \(\qquad \forall x\; \mathit{atLeastAsBig}(x,x)\)

    To model similar things in a modal logic we can think of the states (or worlds) as the individual things. Instead of the first order unary predicate \(\mathit{Red}\,\) we can use a proposition \(\mathit{Red}\). If this proposition holds at a particular state we can think of the state proclaiming "I am red". Similarly for being green.

    The idea that everything is green would correspond to the formula \(\mathit{Green}\) being true at every state. But there is no formula in modal logic that will be true in an arbitrary interpretation if and only if there are two different red things. This is because in the modal logic we have described there is no way to talk explicitly about the states themselves. In first order logic, on the other hand, we have variables and constants that allow us to talk about the individuals in the domain of any interpretation.

    If we can make assumptions about the accessibility relation then more can be expressible. For example, suppose we know the relation \(R\) in a frame \((W,R)\) is the universal relation, \(R = W \times W\), or everything is related to everything. In this case the statement "something is red" would correspond to \(\lozenge \mathit{Red}\) being valid in that particular frame. However, we cannot write a formula in modal logic that forces the accessibility relation to be the universal relation.

    Modal logic is quite limited with respect to what can be said at the level of a particular interpretation. That is the problem of:

    • Given a frame and a valuation can you find a formula that is valid in that interpretation if and only if the frame has certain properties?

    It is known, for example, that you cannot write a formula that detects (by being valid or not) whether a given interpretation has a reflexive accessibility relation. On the other hand, if the valuation is not fixed, so we have a frame but not an interpretation, there is a simple formula which is valid under all valuations for that frame if and only if the frame has a reflexive accessibility relation. One formula that does this is \(p \to \lozenge p\).

    In a similar way to validity in a frame under all valuations being able to express reflexivity, other properties such as being transitive can be handled as well. It is known that this formula

    \(\qquad \qquad \lozenge \lozenge p \to \lozenge p\)

    is valid under all interpretations in a given frame if and only if the accessibility relation \(R\) is transitive. However, it is possible to find a particular valuation for a frame where \(R\) is not transitive but the formula holds at all states in that valuation. This can be done with only three states.

    Going back to the example of \(\mathit{atLeastAsBig}\) above:

    • In first order logic:

      • we have a formula that is true in a given interpretation if the 'at least as big' relation is reflexive:
      • \(\forall x\; \mathit{atLeastAsBig}(x,x)\)

    • In modal logic:

      • we can model the 'at least as big' relation as the accessibility relation, but
      • it is impossible to write a formula that is true at all states in a given interpretation if the 'at least as big' relation is reflexive.

    Modal logic is still valuable in knowledge representation, but it is important to appreciate that there are a lot of subtleties when talking about the ability of different logics to express different kinds of statements.

    3.6 More than one accessibility relation

    Beyond the most basic modal logic you may have several different diamonds \(\langle i \rangle\) (and their dual boxes \([ i ]\)). The semantics then involves several accessibility relations on the same set of 'worlds'.

    There are two cases where this is important in this module:

    1) This is another perspective on description logic. In fact description logic is really just a different notation for a modal logic of this kind, as indicated by these comparisons.

    \[ \begin{array}{ccc} &&\\ \langle \mathsf{hasChild} \rangle \, \mathsf{UnderTen} & \qquad & \exists \,\mathsf{hasChild} . \mathsf{UnderTen}\\[2ex] [ \mathsf{hasChild} ] \, \mathsf{UnderTen} & \qquad & \forall\, \mathsf{hasChild} . \mathsf{UnderTen}\end{array} \]

    2) In modelling time where you can have one relation relating a time to all future times, and another relation relating a time to all past times. We will explore this example further in the final unit.
