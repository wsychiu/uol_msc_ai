    Set-up instructions for virtual machine access

    These instructions outline how to connect to a virtual machine (VM) for this module.

    Warning

    This process will require you to use Duo, the University’s two-factor authentication service. You will need to make sure you have enrolled with Duo with your University account and will be required to use Duo to authenticate your access to the VM.

    Connecting from Linux/MacOS systems

    • Step 1: Open a terminal on your machine.
    • Step 2: Create a directory called .ssh in your home directory (if one doesn’t already exist).
    $ mkdir ~/.ssh
    • Step 3: Create a file named config within your .ssh directory.
    • Step 4: Within this config file include the following contents (ensure that you replace all instances of myusername with your own University username).
    Host *.leeds.ac.uk !remote-access.leeds.ac.uk
ProxyJump myusername@remote-access.leeds.ac.uk
User "myusername"

Host 20.108.153.48
ProxyJump myusername@remote-access.leeds.ac.uk
    • Step 5: Save this file.
    • Step 6: Run the following command in the terminal, again replacing myusername with your University username:
    $ ssh myusername@20.108.153.48
    • Step 7: You’ll then see the following message:
    The authenticity of host 'remote-access.leeds.ac.uk (129.11.78.165)' can't be established.
ECDSA key fingerprint is SHA256:RoIHn1BDSlh/WjGHEG73OFISe6H3q3iLuM4z6tGu29Q.
Are you sure you want to continue connecting (yes/no/[fingerprint])?
    • Step 8: Input yes and then press enter.
    • Step 9: You will then be asked a password. Input your University password.
    • Step 10: Once you have entered your correct password, you will receive the Duo 2 factor authentication prompt as follows:
    $ ssh myusername@20.108.153.48
Password:
Duo two-factor login for myusername@20.108.153.48

Enter a passcode or select one of the following options:

 1. Duo Push to +XX XXXX XXXXXXX
 2. SMS passcodes to +XX XXXX XXXXXX (next code starts with: 2)

Passcode or option (1-2):

    It will be simplest to just generate a passcode instead of using options 1 or 2. Therefore, use your Duo-registered device to generate a passcode and then input the passcode under the above prompt.

    • Step 11: You will then see the following message:
    Warning: Permanently added '20.108.153.48' (ECDSA) to the list of known hosts.
myusername@20.108.153.48's password:
    • Step 12: This time you will need to enter your password but this is NOT your University password. Instead this is a specific password generated for you to use this VM. This specific password will have been emailed to you with a subject of 'Knowledge Representation and Reasoning - Virtual Machine Access'.
    • Step 13: Once you have entered your password, you should then see success message of Welcome to Ubuntu. At the end you will see your command prompt now in green as: 
      myusername@uol-ocom5201m-vm:~$
    • Step 14: It's recommended that you change your VM password once logged in for the first time. In order to do this type passwd and press enter. You will then be asked to enter your current password (supplied to you via email) and then enter your new chosen password.
    • Step 15: Once you have finished with your VM session, type logout into the prompt and press enter. You’ll then be informed that the connection is closed.
    • Step 16: When you wish to reaccess the VM, simply go to the terminal and enter ssh myusername@20.108.153.48 (replacing myusername with your own University username). You will then need to repeat steps 9,10 and 12 in order to log back into the VM.

    Connecting from Windows systems

    Info

    Connecting from Windows requires the installation of an SSH client. We recommend and support users using MobaXTerm.

    Installing MobaXTerm

    • Step 1: Go to the MobXTerm website and select 'Download now' under the Free download section.
    • Step 2: Select the Portable edition. This will prompt a .zip file download. Once downloaded, extract the files in the .zip and select the .exe file. This will open up MobaXTerm.

    Connecting to the VM via MobaXTerm

    • Step 1: Open MobaXTerm and in the top navigation panels, select Session.
    • Step 2: Within the Session window, in the top navigation, select SSH.
    • Step 3: Under the Basic SSH settings area, enter the 'Remote host' as '20.108.153.48'. Select the checkbox next to 'Specify username' and then enter your University username in the associated field.
    • Step 4: Go to the Network settings tab underneath the Basic SSH settings area.
    • Step 5: Select the 'SSH gateway (jump)' button.
    • Step 6: Under 'Gateway host' enter remote-access.leeds.ac.uk. Under 'Username' enter your University username. Then select 'OK'.
    • Step 7: This will take you back to the Network settings tab. Once here, select the 'OK' button at the bottom. This will then prompt a pop-up that asks for your University password. Enter you password and select 'OK'.

    Note

    Please note that the password pop-up may take up to a few minutes to appear.

    • Step 8: After you have correctly entered your password, you will be asked to enter a Duo passcode. Generate a Duo passcode on your Duo-registered device and enter it into the pop-up and select 'OK'.
    • Step 9: You will then be asked if you would like to save your password for remote access. Select 'Yes'. If you do not select 'Yes' you will need to repeat this process each time you access the VM.
    • Step 10: You will then be prompted to create a master password. This is a master password for MobaXTerm and is not related to your VM password. Create something you will be able to remember.
    • Step 11: With the 'User session' sidebar in MobaXTerm, you should now see an item for 20.108.153.48 (myusername). Select this item (you will need to double click).
    • Step 12: This will open up a terminal window for the session and will require you to enter your VM password straight away. This is NOT your University password. Instead this is a specific password generated for you to use this VM. This specific password will have been emailed to you with a subject of 'Knowledge Representation and Reasoning - Virtual Machine Access'.

    Warning

    Please note that you will not be able to copy your password via Ctrl+C/Cmd+C, instead, you will need to right click and select 'Paste'.

    • Step 13: You will then be asked if you wish to save your password for the VM accesss. Select 'yes'.
    • Step 14: The terminal should then open a session into the VM and you will see the prompt as myusername@uol-ocom5201m-vm:~$.
    • Step 15: It's recommended that you change your VM password once logged in for the first time. In order to do this type passwd and press enter. You will then be asked to enter your current password (supplied to you via email) and then enter your new chosen password.
    • Step 16: Once done with the VM, type logout into the prompt and press enter. You’ll then be informed that the connection is closed.
    • Step 17: When revisiting MobaXTerm, select the item underneath 'User session' in the sidebar (you will need to double click). This will reopen the session into the VM. If you receive an error message when trying to reopen the session, please repeat the process from step 1 to reconnect successfully.
